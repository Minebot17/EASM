# Спецификация EASM 0.1

## Машинная модель

- Четыре регистра: `A`, `B`, `C`, `D`.
- Регистры и память содержат знаковые 64-битные целые числа в дополнительном коде.
- Размер памяти задаётся при запуске VM. Прямой адрес имеет вид `[0]`, `[1]` и так далее.
- Косвенный адрес через регистр записывается как `[A]`, а через ячейку-указатель — как `[[0]]`.
- Переполнение оборачивается по модулю `2^64`.
- Деление округляет к нулю. Остаток имеет знак делимого.
- Деление на ноль, неверный адрес, индекс или дескриптор завершают VM со статусом `error`.
- Сдвиги используют младшие 6 бит величины сдвига, то есть диапазон `0..63`.
- Сравнения знаковые и возвращают `0` либо `1`.

## Синтаксис

Одна инструкция занимает одну строку. Операнды разделяются запятыми. Комментарий начинается с `;`.

```asm
label:
mov 0x2a, A       ; hexadecimal immediate
add A, -2, [10]
jnz [10], label
```

Константы поддерживают десятичную, hexadecimal `0x`, binary `0b` и octal `0o` запись. В цифрах можно использовать `_`. Метка при разборе превращается в числовой индекс инструкции; после самомодификации она не перемещается автоматически.

### Адресация памяти

```asm
mov 10, [3]      ; прямая запись в ячейку 3
mov 3, A
mov 10, [A]      ; значение A = 3, запись идёт в ячейку 3
mov 3, [0]
mov 10, [[0]]    ; значение [0] = 3, запись идёт в ячейку 3
```

Косвенная адресация имеет один уровень разыменования. `[A]` читает адрес из регистра `A`; `[[5]]` читает адрес из ячейки `[5]`. Полученный адрес должен попадать в память VM, иначе выполнение завершается с runtime error. Оба вида разрешены как источник и как назначение.

Для большинства бинарных операций применяется форма:

```text
operation left, right, destination
```

Для `div`:

```text
div dividend, divisor, quotientDestination, remainderDestination
```

## Opcode ABI

Индекс является частью формата EASM и используется командой `insert`.

| Индекс | Операция | Операнды | Результат |
|---:|---|---|---|
| 0 | `add` | `left, right, dst` | сложение |
| 1 | `sub` | `left, right, dst` | вычитание |
| 2 | `mul` | `left, right, dst` | умножение |
| 3 | `div` | `left, right, quotientDst, remainderDst` | частное и остаток |
| 4 | `mod` | `left, right, dst` | остаток |
| 5 | `and` | `left, right, dst` | побитовое И |
| 6 | `or` | `left, right, dst` | побитовое ИЛИ |
| 7 | `xor` | `left, right, dst` | побитовое исключающее ИЛИ |
| 8 | `shl` | `value, count, dst` | сдвиг влево |
| 9 | `shr` | `value, count, dst` | арифметический сдвиг вправо |
| 10 | `ushr` | `value, count, dst` | логический сдвиг вправо |
| 11 | `rol` | `value, count, dst` | циклический сдвиг влево |
| 12 | `ror` | `value, count, dst` | циклический сдвиг вправо |
| 13 | `eq` | `left, right, dst` | равно |
| 14 | `ne` | `left, right, dst` | не равно |
| 15 | `lt` | `left, right, dst` | меньше |
| 16 | `le` | `left, right, dst` | меньше либо равно |
| 17 | `gt` | `left, right, dst` | больше |
| 18 | `ge` | `left, right, dst` | больше либо равно |
| 19 | `mov` | `source, dst` | копирование |
| 20 | `neg` | `source, dst` | смена знака |
| 21 | `not` | `source, dst` | побитовое НЕ |
| 22 | `inc` | `source, dst` | увеличение на один |
| 23 | `dec` | `source, dst` | уменьшение на один |
| 24 | `rand` | `dst` | случайный signed int64 |
| 25 | `jmp` | `instructionIndex` | безусловный переход |
| 26 | `jz` | `value, instructionIndex` | переход при нуле |
| 27 | `jnz` | `value, instructionIndex` | переход при ненулевом значении |
| 28 | `nop` | — | пустая операция |
| 29 | `halt` | — | штатная остановка |
| 30 | `insert` | `opcodeIndex, insertionIndex, descriptor...` | вставка инструкции |
| 31 | `delete` | `instructionIndex` | удаление инструкции |
| 32 | `change` | `instructionIndex, descriptor...` | замена всех операндов |

`rand` использует SplitMix64. Seed задаётся извне, обрезается до 64 бит и делает запуск полностью воспроизводимым. Команда генерирует любое значение signed int64 и может писать в регистр или память.

## Самомодификация

`insert`, `delete` и `change` немедленно меняют активный массив инструкций VM. Все индексы относятся к текущему, уже модифицированному коду и начинаются с нуля.

### Дескриптор операнда

Каждый операнд после первых служебных операндов `insert`/`change` вычисляется как указатель `p` на пару ячеек:

| `memory[p]` | Значение `memory[p+1]` | Созданный операнд |
|---:|---|---|
| 0 | `0`, `1`, `2`, `3` | регистр `A`, `B`, `C`, `D` |
| 1 | индекс памяти | `[index]` |
| 2 | int64 | константа |
| 3 | `0`, `1`, `2`, `3` | косвенная память `[A]`, `[B]`, `[C]`, `[D]` |
| 4 | индекс ячейки-указателя | `[[index]]` |

Пример формирования и вставки `mov 42, [10]`:

```asm
mov 2, [0]          ; descriptor 0: immediate
mov 42, [1]
mov 1, [2]          ; descriptor 2: memory reference
mov 10, [3]
insert 19, 7, 0, 2 ; opcode 19 = mov
jmp 7
halt
```

Количество дескрипторов в синтаксисе не ограничено, но полученная инструкция должна удовлетворять своей arity и общим лимитам VM. `change` сохраняет opcode и стабильный внутренний ID инструкции, заменяя весь список операндов.

### Выбор следующей инструкции

Перед выполнением VM запоминает стабильные ID всех инструкций, которые находились после текущей. Если команда не сделала явный переход, следующей становится первая сохранившаяся инструкция из этого прежнего хвоста.

Следствия:

- вставка перед текущей или непосредственно перед её старым преемником не заставляет выполнить вставку немедленно;
- удаление старого преемника передаёт управление следующему сохранившемуся старому преемнику;
- чтобы перейти к только что вставленной команде, можно использовать `jmp`;
- если ни одного старого преемника не осталось, выполнение завершается.

Так выполняется правило: следующей всегда пытается стать инструкция, которая до мутации следовала за текущей, с естественным продолжением при её удалении.

## Ввод и вывод памяти

### Биты

Пробелы и `_` игнорируются. Каждая последовательная группа до 64 бит читается как одно беззнаковое двоичное слово и сохраняется с тем же битовым представлением в signed int64. Первая цифра группы — старший записанный бит.

### UTF-8

Для удобства один UTF-8 байт занимает одну 64-битную ячейку и хранится как `0..255`. При декодировании используется младший байт каждой ячейки; конечные нулевые ячейки отбрасываются. Это не то же представление, что плотный битовый ввод, и в UI форматы выбираются явно.

### Целые числа

Список знаковых чисел через запятые или пробелы напрямую инициализирует последовательные ячейки. Остальная память заполняется нулями.

## Ограничение исполнения

VM всегда получает максимальное количество шагов. Достижение лимита завершает запуск со статусом `limit`. Браузер выполняет VM в Web Worker, поэтому даже большой эксперимент не блокирует редактор и интерфейс страницы.
