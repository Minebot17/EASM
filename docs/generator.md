# Случайный генератор программ

Страница `generator.html` генерирует EASM-программы и выполняет каждую из них на заданной входной памяти. В список результатов попадают только штатно завершившиеся кандидаты, прошедшие выбранный режим сравнения с ожидаемым выходом.

## Настройки

- Формат входа и выхода: UTF-8, биты или список целых чисел.
- Максимальное количество сгенерированных программ: любое положительное целое число. Практическую продолжительность по-прежнему ограничивает лимит времени.
- Количество инструкций в каждом кандидате: `1…64`.
- Размер доступной программе памяти: `1…65536` ячеек. Все сгенерированные прямые адреса и ячейки-указатели находятся в этом диапазоне.
- Лимит общего времени: `0.1…300` секунд.
- Количество параллельных Web Worker: `1…64`; по умолчанию используется до восьми логических процессоров с одним оставленным для интерфейса.
- Seed случайного генератора для воспроизводимых запусков.

На каждый кандидат также устанавливается внутренний лимит шагов. Программы, попавшие в бесконечный цикл, завершившиеся с runtime error или превысившие этот лимит, отбрасываются.

## Режимы сравнения

### Учитывать нули

Сравниваются значения и позиции ячеек. Ожидаемый массив дополняется нулями до размера памяти, поэтому любое дополнительное ненулевое значение считается несовпадением.

### Игнорировать нули

Из фактической и ожидаемой памяти удаляются все нулевые ячейки. Оставшиеся значения должны совпасть по количеству, значению и порядку.

Например, `[0, 10, 0, 20]` проходит фильтр для ожидаемого выхода `[10, 20, 0]`.

## Случайная генерация

- Opcode каждой инструкции равновероятно выбирается из полного реестра EASM, включая переходы и самомодификацию.
- Типы операндов выбираются случайно с соблюдением требований к назначениям.
- Когда требуется операнд памяти, генератор равновероятно выбирает один из трёх видов: прямую ячейку (`[index]`), косвенный указатель через случайный регистр (`[A]`–`[D]`) или косвенный указатель через случайную ячейку (`[[index]]`). На большой выборке каждый вид получает примерно 33%.
- 85% констант выбираются из диапазона `-256…255`, остальные — из полного signed int64. Это сохраняет случайный характер поиска, но позволяет находить практичные программы за разумное время.
- Адреса обычных операндов ограничиваются заданным размером доступной памяти.
- Цели `jmp`, `jz`, `jnz`, `insert`, `delete` и `change` выбираются только среди строк исходного кандидата. Для `change` не смешиваются сдвигающие индексы `insert`/`delete`, поэтому замена всегда соответствует opcode целевой строки.
- Дескрипторы операндов для `insert` и `change` создаются заранее и хранятся в защищённой служебной части памяти. Назначения выбираются только из регистров и трёх допустимых видов памяти — константа назначением стать не может.

Кандидаты распределяются между Worker’ами чередующимися номерами. Seed вычисляется отдельно для номера кандидата, поэтому потоки не повторяют работу, а один и тот же общий seed даёт те же программы независимо от количества Worker’ов. Каждый Worker публикует прогресс каждые 50 программ, поэтому интерфейс остаётся доступным и поиск можно остановить. Для защиты UI сохраняются первые 500 успешных вариантов; общее количество совпадений продолжает отображаться полностью.

Для самомодифицировавшихся кандидатов результат показывает как исходный случайный код, так и код после выполнения.
